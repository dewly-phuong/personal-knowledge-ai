# Query

"Query" refers to an essential operation within an [[wiki/concepts/llm-wiki]] system, functioning as a data processing workflow where an [[wiki/services/llm-agent]] synthesizes answers to user questions by leveraging a structured, persistent, and interlinked knowledge base. This contrasts with traditional [[wiki/concepts/rag]] systems by accumulating knowledge rather than rediscovering it on each query.

## Sequence Steps

1.  **Trigger**: A user initiates the query by asking a question against the [[wiki/concepts/llm-wiki]].
2.  **Retrieval**: The [[wiki/services/llm-agent]] searches for and identifies relevant pages within the wiki. For moderate-scale wikis, this often involves reading the `[[wiki/index.md]]` file to find pertinent entries.
3.  **Synthesis**: The [[wiki/services/llm-agent]] reads the content of the retrieved pages and synthesizes a coherent answer, including citations to the original wiki sources.

## Answer Forms and Persistence

Answers generated by the [[wiki/services/llm-agent]] during a query operation can manifest in various structured forms, adaptable to the user's specific request:

*   Markdown pages
*   Comparison tables
*   Slide decks (potentially using tools like [[wiki/concepts/marp]])
*   Charts (e.g., generated with matplotlib)
*   Canvases

A significant feature of this query pipeline is the ability for valuable answers—such as analyses, comparisons, or newly discovered connections—to be filed back into the [[wiki/concepts/llm-wiki]] as new, persistent pages. This mechanism ensures that user explorations and the insights derived from queries compound the knowledge base, preventing their loss in transient chat history.

## Related Concepts and Services

*   [[wiki/concepts/llm-wiki]]: The overarching system or pattern that defines the knowledge base and its maintenance.
*   [[wiki/services/llm-agent]]: The core service responsible for executing the query process, including retrieval, synthesis, and potential wiki updates.
*   [[wiki/concepts/rag]]: A contrasting method for information retrieval, where knowledge is re-derived per query.
*   `[[wiki/index.md]]`: A catalog file used by the LLM agent to efficiently navigate and locate relevant pages during query processing.
*   [[wiki/concepts/markdown]]: The primary format for content within the wiki and for many query outputs.
*   [[wiki/concepts/obsidian]]: A desktop application often used as an IDE for interacting with the markdown-based wiki.
*   [[wiki/concepts/marp]]: A tool for generating slide decks directly from markdown content, potentially used for query output.
*   [[wiki/concepts/dataview]]: An Obsidian plugin that can query page frontmatter, useful for dynamic tables and lists in wiki output.
*   [[wiki/concepts/git]]: Utilized for version control of the wiki's markdown files, offering history, branching, and collaboration.
