# Project Instructions

## Workflow Conventions

- **Brainstorming First:** Before performing any design, implementation, or decision-making, the `brainstorming` skill MUST be activated to validate the approach, ensure clarity, and document assumptions. No action should be taken without passing through the brainstorming workflow first.

- **Automated Work Logging:** Every time a task involves modifying source code, I MUST log the change in `WORK_HISTORY.md` at the project root. The format for each entry must be: `MM/DD/YYYY - HH-MM-SS | [Model Name] | [Brief Description of Change]`. This must be done in the same conversational turn as the file modification.
